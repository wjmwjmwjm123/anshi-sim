# 安史之乱：中唐续命

对话驱动的联网安史之乱国家经营战役。规则内核不依赖 LLM，人物奏对与史官月报使用 OpenAI 兼容模型。

## 技术栈

- Python 3.13 + FastAPI + Pydantic
- React 19 + TypeScript + Vite
- SQLite WAL（已接入单战役存档与回合日志）
- Pydantic 版本化剧本校验（已接入完整战役 v0.5.0）
- OpenAI 兼容联网模型（LongCat/OpenAI/DeepSeek 等；规则结算不依赖模型）

## 已实现的第一章循环

- 显式事件时钟：敌情复核、将相释疑、出关诏限、退路整备均有起止日与逾期状态
- 军令状态：草拟、核验、发出、执行、完成或拒绝，全链路写入回合结果
- 结构化情报：来源、时效、置信度、来源链独立性和互证状态
- 六项处置：固守、复核、召对、更易监军、整备退路、出关
- 潼关溃败后确定性转入第二章“皇统裂变”，打开长安撤离事件窗

## 已实现的国家经营面

- 七部界面：军情、天下、朝堂、诸军、户部、奏报、史册
- 全国内容：16 地区、13 支跨幕军队、21 名人物、6 个持续事项、18 个事件候选
- 人物召见：近臣入对与边将远奏，回应受身份、公开立场和提问主题约束
- 诏令队列：赈济、征税、补给、动员、筑防、调查、任命、调停，可连续组令和删除
- 统一结算：财政固定流水与诏令按顺序确定性执行，返回逐令报告和字段差异
- 完整存档：潼关战术状态、国家经营状态、待颁诏令和回合日志同时保存
- 史料标识：史实锚定、史料争议、制作估算、虚构设计分别展示
- 五章可推进：每章有时间跨度、强制裁断事件、世界演化、跨章债务与组合结局
- 人物多场景：朝堂公开奏对、密诏私议、外镇远奏使用不同提示和信息边界
- 密诏任务：封缄后跨回合推进，办结形成密奏
- 联网史官：每回合读取已锁定的权威结果生成纪事，模型不能修改数值
- 连续人物对话：保存聊天历史、长期记忆、信任、恩宠、畏惧和皇帝承诺
- 自由诏书：玩家直接写诏文，中书拆成候选执行事项，确认后才进入规则结算
- 战略战争：军队沿路线移动，消耗补给；燕军按章节目标行动；同地自动会战；围城持续推进
- 存档产品流程：主菜单、开始新游戏、继续自动存档、手动另存和载入
- 模型运行审计：记录人物/史官调用的模型、耗时、成功与降级状态

## 联网配置

支持环境变量：

```powershell
$env:OPENAI_API_KEY='...'
$env:OPENAI_BASE_URL='https://api.example.com/v1'
$env:OPENAI_MODEL='model-name'
```

也可点击游戏右上角的模型状态，在界面中配置当前进程。密钥不会通过 API 返回。若仅设置 `LONGCAT_API_KEY`，默认使用 `https://api.longcat.chat/openai/v1` 和 `LongCat-2.0`。

## 启动

```powershell
python apps/api/run.py

cd apps/web
npm.cmd run dev
```

打开 `http://127.0.0.1:5173`。

## 手工验收

1. 重置后直接“奉诏出击”：灵宝溃败，章节转入“皇统裂变”。
2. 重置后“三路复核”再出击：受创退关，可战兵力为 77,000。
3. 重置后依次执行“三路复核、召对释疑、更易监军、整备退路、奉诏出击”：诸军按旗而退，可战兵力至少 88,000。
4. 连续固守四次后再固守：军令被拒绝，日期不推进。
5. 进入“朝堂”，召见高力士并询问钱粮，确认收到人物奏对。
6. 底部依次加入“开仓赈济·长安”和“转运军粮·潼关守军”，点击“颁诏并推进”。
7. 在“史册”查看逐令执行报告；再到“户部/天下/诸军”核对银粮、民心和补给变化。
8. 刷新浏览器，确认回合、诏令结果与国家状态保持。
9. 在人物抽屉切换“朝堂、密诏、远奏”，确认同一问题的措辞和信息边界不同。
10. 完成当前事件裁断并推进若干回合，确认章节、月份、登场人物、军队和持续债务变化。
11. 返回主菜单，验证自动存档继续；另存一个命名存档，再载入恢复。
12. 连续召见同一人物三次，确认近期对话、关系和承诺写入朝堂侧栏。
13. 在朝堂直接写自由诏书，核对中书拆解结果，确认后再推进回合。
14. 在“诸军”调动唐军，推进回合后查看燕军行动和会战记录。

每次发令自动保存到 `data/anshi.db`，重启后恢复；旧版 `0.1.0` 存档在载入时自动补齐新字段。
